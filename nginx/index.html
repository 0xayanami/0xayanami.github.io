nginx

nginx
Publish Date:   2022-11-18

Nginx是一款轻量级的Web服务器,Nginx采用了事件驱动模型，每个连接不再会占据一个进程/线程的资源，一般来说只会消耗文件句柄和一些其他必要的资源。每个连接的低内存消耗自然就大大提升了并发量（10000个非活跃的HTTP keepalive连接大约占据2.5MB内存，单机并发量达到10万以上）。

Nginx的Master-Worker模式

启动Nginx后，其实就是在80端口启动了Socket服务进行监听

Master进程的作用是？

读取并验证配置文件nginx.conf；管理worker进程

Worker进程的作用是？

每一个Worker进程都维护一个线程（避免线程切换），处理连接和请求

Nginx如何做到热部署？

修改配置文件nginx.conf后，重新生成新的worker进程，当然会以新的配置进行处理请求，而且新的请求必须都交给新的worker进程，至于老的worker进程，等把那些以前的请求处理完毕后，kill掉即可

Nginx如何做到高并发下的高效处理？

Nginx采用了Linux的epoll模型，epoll模型基于事件驱动机制，它可以监控多个事件是否准备完毕，如果OK，那么放入epoll队列中，这个过程是异步的。worker只需要从epoll队列循环处理即可

configure

// cd到源代码目录
// 生成makefile文件
./configure --prefix=/usr/local/nginx
// 编译链接.c
make
// 将生成的文件移到合适位置
sudo make install

nginx使用流程

cd $prefix
// 启动
sbin/nginx
// 关闭
sbin/nginx -s quit

默认配置文件

// 指定worker进程数，cpu核心数的1~2倍
worker_processes  1;
events {
    // 一个worker进程同时能支持的最大连接数
    worker_connections  1024;
}
http {
    // 把另一个文件包含进来
    include       mime.types;
    // 默认的响应类型
    default_type  application/octet-stream;
    // 调用sendfile函数传输文件
    sendfile        on;
    // 及复用tcp连接的超时时限
    keepalive_timeout  65;
    // 虚拟服务端
    server {
        listen       80;
        server_name  localhost;
        // 当有URL的路径部分匹配 / 时，
        // 找到服务端的html目录，
        // 访问其中的index.html文件
        location / {
            root   html;
            index  index.html index.htm;
        }
        // HTTP的响应是500 502 503或者是504时，
        // 将URL的路径部分改写成 /50x.html 
        error_page   500 502 503 504  /50x.html;
        // 当有URL的路径部分匹配 /50x.html 时，
        // 访问服务端的html目录中的50x.html文件
        location = /50x.html {
            root   html;
        }
    }
}
